# Skill: Format Working Note

**Purpose:** Maintain one current, actionable working note for an active project.

## Core rules

1. Treat plans as provisional, not settled fact.
2. Preserve every useful idea and reference.
3. Convert next steps into checkboxes.
4. Turn requested changes into ready-to-run AI prompts when useful.
5. Ask before deleting or overwriting unrelated material.

## Current-note tracking

1. Save working notes under `Efforts/<Project>/`.
2. Give the newest live note `status: current`.
3. Mark the previous current note `status: superseded` and link both notes.
4. Keep exactly one current working note per project.
5. Update `_<Project> (Project Home).md` to point to the current note.

## Suggested sections

- Where things stand
- Next steps
- Prompts for AI
- Ideas / parking lot
- References
- Open questions / might change
