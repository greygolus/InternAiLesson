# Skill: Format Rough Notes

**Purpose:** Turn raw notes in `Rough Notes/` into useful, lightly formatted notes.

## Steps

1. Read the vault maps and the user's context.
2. Process every Markdown note in `Rough Notes/` except names beginning with `_`.
3. Route durable knowledge to `Atlas/`, time-based material to `Calendar/`, and active project work to `Efforts/`.
4. If a note is project planning, follow `AIOS/skills/Format Working Note.md`.
5. Preserve meaning and voice. Fix only clear typos and structure.
6. Apply exactly one domain tag using `AIOS/Tag Map.md`.
7. Keep useful references and verify `[[wikilinks]]`.
8. Write the formatted note before removing the rough original.
9. Ask when the destination or domain is genuinely unclear.

Report what moved, where it moved, and anything left for clarification.
